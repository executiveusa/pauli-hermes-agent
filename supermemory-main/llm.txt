# Supermemory - AI Second Brain & Knowledge Management

## Purpose
Supermemory is a persistent knowledge management system that functions as an AI second brain, enabling agents to save, search, and retrieve information from URLs, PDFs, plain text, and connected services (Notion, Google Drive, OneDrive). It provides MCP integration for seamless memory access across Claude, Cursor, and other AI tools.

## Capabilities
- **Universal Memory Capture**: Save content from URLs, PDFs, text files, screenshots
- **Semantic Search**: Natural language queries across all stored memories
- **MCP Integration**: Direct access from Claude Desktop, Cursor, Windsurf, and MCP-compatible tools
- **Browser Extension**: One-click save from ChatGPT, Claude web, Twitter/X
- **Raycast Extension**: Quick capture and search from macOS
- **Service Connectors**: Sync with Notion, Google Drive, OneDrive, Dropbox
- **Chat Interface**: Conversational queries against memory database
- **Tagging & Organization**: Automatic and manual categorization
- **Embeddings-Based Retrieval**: Context-aware semantic matching
- **Self-Hosting**: Full control over data and privacy
- **Export/Import**: Portable memory archives
- **API Access**: Programmatic memory management

## Dependencies
- **Node.js 18+**: Runtime environment
- **PostgreSQL 14+**: Primary database with pgvector extension
- **Redis** (optional): Caching layer for faster searches
- **OpenAI API** or **Local Embeddings**: Text embedding generation
- **pgvector Extension**: Vector similarity search in PostgreSQL
- **Docker** (recommended): Containerized deployment
- **MCP Server**: For Claude/Cursor integration

## Usage
**For AI Agents:**
```typescript
import { Supermemory } from 'supermemory-sdk';

// Initialize client
const memory = new Supermemory({
  apiKey: process.env.SUPERMEMORY_API_KEY,
  endpoint: 'http://localhost:8087'
});

// Save memory
await memory.save({
  content: "Meeting notes: Discussed Q1 roadmap priorities...",
  source: "meeting-2026-02-02",
  tags: ["meetings", "roadmap", "q1-2026"],
  metadata: {
    date: "2026-02-02",
    participants: ["Alice", "Bob"]
  }
});

// Search memories
const results = await memory.search(
  "What were the Q1 priorities?",
  { limit: 5, tags: ["roadmap"] }
);

// Get similar memories
const related = await memory.findSimilar(memoryId, { limit: 10 });
```

**CLI Commands:**
```bash
# Save URL
supermemory save https://example.com/article --tags "research,ai"

# Save file
supermemory save document.pdf --tags "contracts,legal"

# Search
supermemory search "customer feedback about pricing"

# List recent
supermemory list --limit 20 --sort recent

# Export archive
supermemory export --output backup.json --since "2026-01-01"
```

## API/Interface
**REST API Endpoints:**
```
POST /api/memories                    - Save new memory
GET  /api/memories                    - List memories
GET  /api/memories/:id                - Get specific memory
PUT  /api/memories/:id                - Update memory
DELETE /api/memories/:id              - Delete memory
POST /api/search                      - Semantic search
POST /api/similar/:id                 - Find similar memories
POST /api/chat                        - Chat with memories
GET  /api/tags                        - List all tags
POST /api/import                      - Import memories
GET  /api/export                      - Export memories
```

**Example 3: Customer Context Persistence**
```typescript
// OpenClaw stores user preferences and history
async function saveCustomerInteraction(userId: string, message: string) {
  await memory.save({
    content: `User ${userId}: ${message}`,
    tags: ["customer", userId, "chat-history"],
    metadata: {
      timestamp: new Date(),
      channel: "whatsapp",
      sentiment: await analyzeSentiment(message)
    }

// Retrieve customer context before responding
async function getCustomerContext(userId: string) {
  const history = await memory.search(
    `conversations with ${userId}`,
    { tags: [userId], limit: 10 }
  );
  return history.map(m => m.content).join('\n');
}
```

## Configuration
**Memory Settings:**
```yaml
storage:
  database_url: "postgresql://localhost/supermemory"
  vector_dimension: 1536  # OpenAI ada-002
  
embeddings:
  provider: "openai"  # or "local", "cohere"
  model: "text-embedding-3-large"
  batch_size: 100

search:
  default_limit: 10
  similarity_threshold: 0.7
  rerank: true
  
import:
  allowed_sources: ["url", "pdf", "txt", "md", "docx"]
  max_file_size: "50MB"
  auto_summarize: true
```

## Advanced Features
**Auto-Tagging:**
```typescript
// Automatic tag extraction
await memory.save({
  content: "Discussed React Server Components in Next.js 15",
  auto_tag: true  // Extracts: ["react", "next.js", "server-components"]
});
```

**Memory Chains:**
```typescript
// Link related memories
const parent_memory = await memory.save({...});
await memory.save({
  content: "Follow-up discussion",
  parent_id: parent_memory.id
});

// Get memory thread
const thread = await memory.getThread(parent_memory.id);
```

**Scheduled Summaries:**
